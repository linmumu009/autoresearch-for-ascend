# v0.1.0 Update Note

Date: 2026-07-02

## Summary

This version creates the first Ascend-specific autoresearch prototype.

The project is derived from the idea of Karpathy's `autoresearch`: an agent
edits a constrained experiment file, runs a fixed-budget training job, compares
a fixed metric, and keeps only improvements.

For our environment, the original loop is adapted to:

- Huawei Ascend 910C server hardware,
- an isolated `llin-autoresearch` Docker container,
- local Qwen3-0.6B model weights,
- China-accessible package mirrors,
- a single visible NPU for the first prototype.

## What Changed

- Created `ascend_autoresearch/prepare.py` as the fixed data and validation layer.
- Created `ascend_autoresearch/train.py` as the agent-editable experiment file.
- Created `ascend_autoresearch/program.md` as the experiment protocol.
- Created `ascend_autoresearch/run_train.sh` to source Ascend runtime environment.
- Ran an initial 5-minute baseline and three gradient-accumulation experiments.

## Difference From Original Autoresearch

The original project trains a small GPT-style model from scratch for a fixed
5-minute budget. This prototype fine-tunes Qwen3-0.6B on Ascend NPU hardware.

The 5-minute budget is kept as an exploration budget, not as a final proof. For
serious decisions, promising 5-minute candidates should be confirmed with a
longer run.
