# v0.2.0 Update - MindSpeed-LLM Framework Adapter

Date: 2026-07-02

## What Changed

- Added a MindSpeed-LLM smoke script for Qwen3-0.6B full SFT on one visible NPU.
- Added a deepscaler-to-Alpaca preparation script for MindSpeed-LLM SFT data.
- Added a framework evaluation document focused on can-run, efficiency, and
  effect.
- Updated the root README and changelog with the MindSpeed-LLM evaluation path.

## Why

The first HF + `torch_npu` prototype proved that the core autoresearch loop can
run on the Ascend server. The next question is whether the same idea benefits
from an Ascend-native acceleration framework. MindSpeed-LLM is the most direct
candidate for Qwen3-0.6B text-only SFT.

## Current Status

- HF thin loop: runs and has a best recorded validation loss of `6.127654`.
- MindSpeed-LLM: imports successfully in the isolated container with a matched
  MindSpeed checkout and completes a 3-step full-SFT smoke on deepscaler-derived
  data.
- MindSpeed-MM: kept as reference, but deferred for this text-only model path.

## Smoke Result

- Deepscaler-derived MindSpeed-LLM SFT data: 512 records.
- Step times: first step `1729.6 ms`, then `249.9 ms` and `238.9 ms`.
- Train loss: `1.0280 -> 0.9771 -> 0.6329`.
- Allocated HBM after first iteration: about `10.3 GB`.
- Strict validation-loss comparison with the HF thin loop is still pending.
