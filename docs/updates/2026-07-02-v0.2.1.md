# v0.2.1 Update - MindSpeed-LLM Validation Smoke

Date: 2026-07-02

## What Changed

- Added configurable validation controls to the MindSpeed-LLM smoke script:
  `DATA_SPLIT`, `EVAL_INTERVAL`, `EVAL_ITERS`, and `SAVE_INTERVAL`.
- Recorded the first held-out MindSpeed-LLM validation-loss run.

## Result

Run configuration:

- Qwen3-0.6B full SFT
- 512 deepscaler-derived Alpaca-style SFT records
- `DATA_SPLIT=90,10,0`
- `TRAIN_ITERS=6`
- `EVAL_INTERVAL=3`
- `EVAL_ITERS=2`

Observed validation:

- iteration 3 validation loss: `1.254728`
- iteration 6 validation loss: `0.691403`
- final validation-set loss after checkpoint save: `0.611867`

## Interpretation

MindSpeed-LLM now has a repeatable validation signal in our isolated Ascend
container. This is enough for framework-local autoresearch decisions. It is not
yet directly comparable to the HF thin-loop validation loss, because the HF
baseline evaluates raw next-token text while this MindSpeed run evaluates
Alpaca-style SFT labels.
