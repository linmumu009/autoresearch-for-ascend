## Interpretation

This is the best fixed raw-HF validation result so far, improving the runner
baseline raw HF val_loss by `2.581577` and the base Qwen3-0.6B raw HF val_loss
by `2.596328`.

The result still supports the short-budget autoresearch idea: with a fixed
train -> convert -> evaluate loop, changing only the candidate env file can
find measurable improvements. However, the MindSpeed-local validation and train
losses are rising at these high learning rates, so this does not mean
`1.0e-4` is safe for longer training. The next step is to bracket the 6-step
boundary with a small higher-LR probe, then confirm the best region under a
longer budget.
