## Runtime Note

A plain `docker exec` launch loaded an incompatible
`/usr/local/Ascend/ascend-toolkit/latest/lib64/libruntime_common.so` and failed
to import `torch_npu`. Prepending
`/usr/local/Ascend/cann-9.0.0/aarch64-linux/lib64` fixed the import inside the
current process, so the runner now applies that path preference automatically
when the directory exists.

## Interpretation

`LR=1.1e-4` is the current best on the fixed raw-HF validation surface,
improving the runner baseline raw HF val_loss by `2.691313` and the base
Qwen3-0.6B raw HF val_loss by `2.706064`.

The result supports continuing the 6-step boundary search, but the rising
MindSpeed-local validation and train losses still warn that high LR candidates
must be bracketed and then confirmed under a longer budget.
