# v0.3.3 - Bilingual README And Higher LR Boundary Search

Date: 2026-07-03

## What Changed

- Added `README.zh-CN.md`.
- Added language links between `README.md` and `README.zh-CN.md`.
- Added `framework_adapters/mindspeed_llm/candidates/lr_1p5em5_6step.env`.

- Updated the MindSpeed-LLM adapter notes, framework evaluation, changelog, and
  main README files with the new candidate results.

## Why

The user-facing GitHub page now needs both English and Chinese entry points.
The v0.3.2 result also showed that `LR=1.0e-5` was still improving, so this
version probes whether the 6-step search starts to over-shoot at `1.5e-5` or
`2.0e-5`.

## Results

| Candidate | LR | Raw HF Val Loss | Raw HF Val PPL | MindSpeed Valid Loss | Last Train Loss |
| --- | ---: | ---: | ---: | ---: | ---: |
| `lr_1p5em5_6step.env` | `1.5e-5` | 14.061219 | 1278526.649 | 0.329362 | 0.547393 |
| `lr_2em5_6step.env` | `2.0e-5` | 13.792945 | 977686.773 | 0.310141 | 0.513032 |

## Decision

Keep `lr_2em5_6step.env` as the current best 6-step MindSpeed candidate.

Compared with the runner baseline raw HF val_loss `14.962966`, it improved by
`1.170021`. Compared with base Qwen3-0.6B raw HF val_loss `14.977717`, it
improved by `1.184772`.

## Next Step

The 6-step over-shoot boundary is still above `2.0e-5`. The next experiment
should either cautiously test a higher LR such as `3.0e-5`, or verify
`2.0e-5` on a longer budget before continuing the LR climb.
