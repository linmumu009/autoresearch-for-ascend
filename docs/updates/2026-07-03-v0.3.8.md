# v0.3.8 - 12-Step Boundary Refinement And 7e-5 Probe

Date: 2026-07-03

## What Changed

- Added `framework_adapters/mindspeed_llm/candidates/lr_4p5em5_12step.env`.
- Added `framework_adapters/mindspeed_llm/candidates/lr_7em5_6step.env`.
- Updated English and Chinese README files, framework evaluation notes,
  changelog, and MindSpeed-LLM adapter notes with the new results.

## Why

The v0.3.7 run showed that 12-step performance worsened at `5.0e-5` compared
with `4.0e-5`, while the 6-step search continued improving at `6.0e-5`. This
version checks the midpoint `4.5e-5` for the 12-step boundary and probes
`7.0e-5` for the short-budget boundary.

## Results

| Candidate | LR | Steps | Raw HF Val Loss | Raw HF Val PPL | MindSpeed Valid Loss | Last Train Loss |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `lr_4p5em5_12step.env` | `4.5e-5` | 12 | 12.678998 | 320936.202 | 0.334108 | 0.503089 |
| `lr_7em5_6step.env` | `7.0e-5` | 6 | 12.459524 | 257692.972 | 0.365993 | 0.577371 |

## Decision

Keep `lr_7em5_6step.env` as the best observed candidate overall.

For 12-step runs, `4.5e-5` is still worse than `4.0e-5`, so the tested local
best for that budget remains `4.0e-5`. For 6-step runs, the boundary still has
not over-shot at `7.0e-5`.

## Next Step

Probe `8.0e-5` on the 6-step boundary. If that still improves, continue upward;
if it worsens, bracket the short-budget optimum between `7.0e-5` and `8.0e-5`.
