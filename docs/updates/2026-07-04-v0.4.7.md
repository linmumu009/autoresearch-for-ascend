# v0.4.7 - 2026-07-04

This iteration trained and saved the `1.0e-4` ms-swift DPO LoRA adapter for
100 steps, then checked it on the independent held-out pair set and on 32
held-out generation prompts.

## Notes

- Extending `1.0e-4` from 50 to 100 steps increases held-out margin from
  `2.3070` to `2.4882`.
- The extra steps make the mean chosen delta slightly negative
  (`+0.0856` -> `-0.0208`), so the 50-step adapter is still the cleanest
  conservative setting.
- The `2.0e-4` 50-step adapter remains the maximum-margin setting, but mostly
  by suppressing rejected responses.
- Greedy generation still has not moved: 32/32 held-out prompt outputs matched
  the base model exactly.
